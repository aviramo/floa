# FLOA

מנוע שבונה אתרים סטטיים (HTML/CSS/JS), ועסק אחד או יותר שנבנים ממנו. FLOA עצמו מתארח ב-GitHub Pages על `floa.co.il`.

## שתי שכבות, וזה כל הסיפור

**`src/` הוא המנוע.** רכיבים, לייאאוט, מערכת העיצוב, עוזרים. אין לו מותג, אין לו דומיין, אין לו מספר טלפון ואין לו קופי. הוא יודע לרנדר hero. הוא לא יודע של מי.

**`businesses/<key>/` הוא עסק.** הזהות שלו (`content/site.js`: מותג, `origin`, צבעים, WhatsApp, נמען הלידים), הקופי שלו, רשימת הדפים שלו, וה-`theme.css` שלו. FLOA הוא פשוט הראשון שבהם, `businesses/floa/`.

**אם קובץ ב-`src/` מזכיר עסק בשם, משהו נשבר.** העסק מגיע דרך `ctx.site`, לא דרך import.

כל עסק נבנה לאתר משלו: דפים, `css/styles.css`, `js/main.js`, `sitemap.xml` ו-`llms.txt` משלו. עסק אף פעם לא מעורבב בתוצר של אחר.

## הדף נבנה, לא נערך

קבצי ה-HTML (`index.html`, `digital-products/index.html`, וכן תיקיית כל עסק) הם **תוצרי בילד**. אין לערוך אותם ישירות, כל עריכה בהם תימחק בבילד הבא.

- **מקור האמת:** `src/` (המנוע) ו-`businesses/` (העסקים)
- **לבנות:** `node build.mjs`, או `node build.mjs --only=floa` לעסק אחד
- **לבדוק:** `npm test` (ניתוב הלידים ב-Worker)
- הבילד מייצר גם `robots.txt`, `sitemap.xml`, `llms.txt`, את `worker/src/businesses.js`, ומחשב hash לשבירת קאש (`css/styles.css?v=…`) לתוך כל דף

אחרי כל שינוי ב-`src/` או ב-`businesses/` צריך להריץ `node build.mjs` לפני קומיט, אחרת האתר החי לא יתעדכן.

## עסק חדש

תיקייה ב-`businesses/`, ובה `index.js` (המניפסט: `key`, `out`, `lead`), `content/site.js` (הזהות), `content/…` (הקופי), `pages/` (הדפים) ו-`theme.css`. אפשר להעתיק את `businesses/floa/` ולקצץ.

- **`out`** היא התיקייה שאליה הוא נבנה, ולכן גם הכתובת: `out: "dana"` נותן `floa.co.il/dana/`. עסק עם דף אחד מוציא `index.html` בלבד; עסק עם עוד דפים מוציא גם `dana/summer/`. אותו מנגנון, בלי מקרה קצה.
- **`origin`** ב-`content/site.js` הוא מה שהופך אותו לאתר עצמאי כשיהיה לו דומיין. הכתובת היא הגדרה, לא ארכיטקטורה.
- **`theme.css`** נכנס **אחרון** במפל, אחרי כל הרכיבים, ולכן הוא תמיד מנצח. מדריסת טוקנים בלבד ועד שינוי מלא של איך נראה `card` אצל אותו עסק.
- **`lead.to`** הוא **שם של משתנה סביבה** ב-Worker, לא כתובת מייל. הריפו ציבורי. אחרי הגדרת העסק: `npx wrangler secret put LEAD_TO_DANA && npx wrangler deploy`, אחרת כל פנייה תיפול ב-502.

## כל דף חדש נבנה מהרכיבים הקיימים

**דף חדש לא מביא איתו רכיבים משלו כל עוד יש רכיב קיים שעושה את העבודה.** הוא מרכיב את עצמו ממה שכבר יש ב-`src/components/`: `hero`, `section`, `section-head`, `card`, `button`, `whatsapp-button`, `timeline`, `faq`, `about`, `project-card`, `contact`, `cta-band`, `cta-dock`, `urgency` וכו'.

- **הטופס:** יש טופס אחד, `contact`, ובו שם וטלפון בלבד וכפתור WhatsApp מתחתיו. כל דף שאוסף לידים משתמש בו, עם הכותרת שלו, בלי טופס משלו ובלי שדות נוספים.
- **WhatsApp:** רק דרך רכיב `whatsapp-button`. אסור לכתוב ידנית `<a class="btn btn-wa">`, כדי שהמספר, ההודעה והמראה לא יתפצלו.
- **בלי HTML ידני בדף:** אם בקובץ `businesses/*/pages/*.js` נכתב markup שמשתמש במחלקות CSS של רכיב קיים, זה סימן שהיה צריך לקרוא לרכיב.
- **מותר להוסיף רכיב חדש** רק כשבאמת אין קיים שמתאים, ואז הוא נכנס ל-`src/components/` ול-`src/bundle.js` וזמין לכל העסקים. **רכיב הוא של המנוע, לא של עסק:** אם הוא נכון רק לעסק אחד, מקומו ב-`theme.css` שלו.
- **ייחודי לעסק:** הקופי (`businesses/<key>/content/…`), סדר הסקשנים, וה-CSS. הרכיבים משותפים.

עסק מייבא מהמנוע דרך `#components/…`, `#layouts/…` ו-`#lib/…` (מוגדרים ב-`package.json`), אף פעם לא דרך `../../src/`.

## גיט: לדחוף ל-main בכל פעם

**בסיום כל שינוי צריך לקמט ולדחוף ל-`main` בלי לחכות לבקשה.** אין צורך לשאול רשות.

```
node build.mjs && git add -A && git commit -m "…" && git push origin main
```

`main` הוא הענף שממנו GitHub Pages מגיש את האתר, ולכן פוש ל-main = עלייה לאוויר. אין ענפי פיצ'רים ואין PR-ים לשינויים רגילים.

לפני פוש תמיד לעשות `git pull --rebase`. סוכנים בענן פותחים לפעמים PR-ים שמתמזגים ל-main ומשאירים את המקומי מאחור.

### התנגשות בקבצי HTML היא כמעט תמיד רעש

אם `git pull` נחסם בגלל קבצי HTML, סביר שההבדל היחיד הוא ה-hash של הקאש. לזרוק את התוצרים ולבנות מחדש:

```
git checkout -- '*.html' && git pull --rebase && node build.mjs
```

**לוודא לפני שזורקים** שאין שם עבודה אמיתית: `git diff -- '*.html'`.

## שפה

הדף כולו עברית, `dir="rtl"`. טקסט מול לקוח נכתב בעברית טבעית, לא תרגום מאנגלית.

### בלי מקף ארוך

**אסור להשתמש במקף ארוך (em dash) `—` או במקף בינוני (en dash) `–` בשום טקסט שמגיע לדף.** זה כולל את הקופי, כותרות, `alt`, תיאורי meta ו-JSON-LD, וגם הערות HTML שנשלחות לדפדפן.

הכלל הוא לא להחליף את התו במקף רגיל, אלא **לנסח מחדש**: פסיק, נקודה או נקודתיים כמעט תמיד עושים את העבודה טוב יותר בעברית.

```
✗ מחדדים את המסר ומחברים את התהליכים — מדף נחיתה ועד מערכת
✓ מחדדים את המסר ומחברים את התהליכים, מדף נחיתה ועד מערכת

✗ ממשק צפוף או איטי, ובמובייל — שם רוב המשתמשים — עוד יותר
✓ ממשק צפוף או איטי. במובייל, שם נמצאים רוב המשתמשים, זה מורגש עוד יותר
```

המקף העברי (מקף) `־` שבתוך מילים כמו `ב־WhatsApp` הוא תקין ונשאר.

לבדיקה אחרי בילד:

```
grep -c "—\|–" *.html */index.html
```
